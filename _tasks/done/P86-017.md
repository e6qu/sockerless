# P86-017: Regression coverage for BUG-692 in CI

**Status:** Done
**Branch:** `phase86-complete-runner-support`

## What

Extended the ECS smoke test so `docker run --rm alpine echo …` runs on every PR. That path exercises the create + attach + start + wait flow that BUG-692 broke. Missing from the prior smoke coverage, which only used `docker create` followed by `docker start` (no attach).

## Files

| File | Change |
|------|--------|
| `smoke-tests/run.sh` | Added a `docker run --rm alpine echo "hello from attach"` step after the existing create/start/stop/rm coverage. Fails the job if the hijacked attach stream times out or returns the wrong bytes |

## Why this catches BUG-692

The broken behavior: `POST /containers/{id}/attach` returned an immediately-EOF pipe, so the docker-CLI-side attach goroutine saw no bytes and never signalled the main goroutine to proceed to `POST /start` — hence the hang. `docker run --rm` exercises exactly this sequence. The new smoke step times out at 60s and asserts the expected stdout line, so a regression surfaces immediately in CI instead of at live-AWS session time.

## Why this catches BUG-693

Less directly — the smoke test runs against the simulator where pull-through-cache APIs aren't implemented, so `resolveImageURI` degrades to the raw ref. Simulator's ECS-RunTask then hands the raw `alpine` to the host's Docker socket which pulls normally. So BUG-693 stays hidden in simulator smoke. Live AWS session 2 is the only place BUG-693 gets exercised end-to-end; unit tests in `image_resolve_test.go` cover the shape logic.

## Out of scope

- Extending to `cloudrun` and `aca` smoke tests — same class of bug could affect them. Flagged as a follow-up because ensuring each simulator emits log streams in the right shape is per-backend work.
- Running `SOCKERLESS_INTEGRATION=1 go test ./backends/ecs/...` in CI (which would run the Go integration tests). Skipped for now — the smoke test already provides equivalent end-to-end coverage, and the integration tests have heavy build prerequisites (eval-arithmetic binary, etc.). The unit tests added in P86-015/016 run on every PR and cover the code-shape side.

## Tests

`bash -n smoke-tests/run.sh` clean. Manual simulation of the test flow confirms it would catch a regression: if `ContainerAttach` returns the old EOF pipe, docker CLI waits for bytes, times out at 60s, and the assertion fails with "FAIL (timed out or errored)".
