# P86-007: SaaS docs for ECS + github.com + gitlab.com

**Status:** Done
**PR:** (pending — same branch)
**Branch:** `phase86-complete-runner-support`

## What

Three new docs covering what's needed to use sockerless ECS backend with the real `actions/runner` and `gitlab-runner` binaries registered against `github.com` / `gitlab.com`:

- **`docs/ECS_LIVE_SETUP.md`** — Terraform pointer, backend startup env vars, endpoint-exposure options (SSH tunnel / ALB+mTLS / Cloudflare Tunnel / VPN), Docker Hub rate-limit notes, verification against the live-mode smoke tests, cost guardrails, common troubleshooting.
- **`docs/GITHUB_RUNNER_SAAS.md`** — registering `actions/runner` on real github.com, `DOCKER_HOST`+mTLS wiring, a smoke workflow covering the three shapes (shell / `container:` / `services:`), per-feature support matrix, troubleshooting.
- **`docs/GITLAB_RUNNER_SAAS.md`** — same shape for `gitlab-runner` on real gitlab.com via the docker executor, with the `runners.docker.host` field pointed at sockerless; smoke pipeline; per-feature matrix.

The existing `GITHUB_RUNNER.md` and `GITLAB_RUNNER_DOCKER.md` got a pointer to the new SaaS docs so a reader landing on either entry isn't surprised by the act / self-hosted-GitLab-CE scope.

## Caveats

The three flows are not yet validated end-to-end — that's blocked on AWS credentials and real SaaS tokens (Phase 86 AWS track). Each doc has an explicit "Phase 86 AWS track" section calling out that status, so an operator reading them knows what's been tested vs. designed.
