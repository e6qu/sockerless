# P43-008: Local Resource Registry + REST Endpoints

**Status:** Done
**Phase:** 43 — Cloud Resource Tracking

## What
Created `ResourceRegistry` for tracking all cloud resources:
- `ResourceEntry` struct: ContainerID, Backend, ResourceType, ResourceID, InstanceID, CreatedAt, CleanedUp
- `ResourceRegistry` with sync.RWMutex, entries map, JSON file persistence
- Methods: NewResourceRegistry, Register, MarkCleanedUp, ListActive, ListOrphaned, ListAll, Save, Load

REST endpoints on BaseServer:
- `GET /internal/v1/resources` — list all (filter `?active=true`)
- `GET /internal/v1/resources/orphaned?max_age=1h` — orphaned resources
- `POST /internal/v1/resources/cleanup` — trigger cleanup (requires CloudScanner)

All 6 backends register resources after cloud create and mark cleaned up on remove.

## Tests
5 unit tests: register+list, mark cleaned up, list orphaned, save/load roundtrip, load non-existent file.
