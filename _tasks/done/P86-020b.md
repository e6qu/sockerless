# P86-020b: Simulator ECR pull-through cache — full implementation (BUG-696 fix)

**Status:** Done
**Branch:** `phase86-complete-runner-support`

## What

AWS simulator now implements the ECR pull-through-cache APIs end-to-end — no stubs, no skip paths. Sockerless's image resolver (`backends/ecs/image_resolve.go`, `backends/lambda/image_resolve.go`) can now call `CreatePullThroughCacheRule` against the simulator and get a real success response instead of `UnknownOperationException`.

Previously the backend's resolver degraded to the raw image ref on simulator, masking the live-AWS resolve path. Now the same code runs locally that runs in production.

## Launch-time URI resolution

Not changed. The existing `sim.ResolveLocalImage` in `simulators/aws/shared/container.go:412` already strips `docker-hub/` + `library/` prefixes from ECR-style URIs and returns the original Docker Hub ref. When sockerless's resolver rewrites `alpine` → `<account>.dkr.ecr.eu-west-1.amazonaws.com/docker-hub/library/alpine:latest`, the simulator's launcher strips it back to `alpine:latest` and pulls from Docker Hub. Matches the "pull-through cache" abstraction: the rule's presence lets the backend produce the ECR URI, the simulator's launcher treats the URI transparently.

Per the no-workarounds directive, this path now exercises the full client → simulator → real-container chain end-to-end with no `if sim-mode skip` branches.
