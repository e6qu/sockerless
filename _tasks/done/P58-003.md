# P58-003: Production Dockerfiles

Created 3 new production Dockerfiles:
- `bleephub/Dockerfile.release` — builds bleephub server, context is repo root
- `frontends/docker/Dockerfile` — builds frontend, copies api/agent/backends/core
- `backends/memory/Dockerfile` — builds memory backend, copies api/agent/sandbox/backends/core

All use multi-stage build pattern: `golang:1.24-alpine` builder, `alpine:3.20` runtime, `CGO_ENABLED=0`, `GOWORK=off`.

Bumped 3 simulator Dockerfiles from `golang:1.23-alpine` to `golang:1.24-alpine`.
