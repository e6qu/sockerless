# P86-001: Drop `-wasm` / `-faas` variant routing

**Status:** Done
**PR:** (pending)
**Branch:** `phase86-pr1-drop-wasm-routing`

## What

Removed the `get_test_variant` remapping layer from the E2E runner harnesses. The remap pointed `services` / `custom-image` / `container-action` to `-wasm` / `-faas` variant files for FaaS-leaning backends. After commit `daeff00` removed the variant `.yml` files, the remap silently pointed at non-existent files; stale summary logs carried the last green results from before the deletion. The harnesses now run the real pipeline names directly and fail honestly when a file is missing.

## Files

| File | Change |
|------|--------|
| `tests/e2e-live-tests/lib.sh` | Removed `get_test_variant()` and `should_skip_for_faas()`. `uses_reverse_agent()` kept — still used by `start-backend.sh` for callback URL setup |
| `tests/e2e-live-tests/github-runner/run.sh` | Dropped `VARIANT=$(get_test_variant ...)`; workflow file path now derived from `$wf` directly |
| `tests/e2e-live-tests/gitlab-runner-docker/run.sh` | Dropped variant mapping loop; `PIPELINE_ARG` is now the space-separated `ALL_PIPELINES` joined with commas |

## Details

- Memory backend and all reverse-agent backends now run the same test names (no per-backend rewriting).
- Missing pipeline/workflow files produce a FAIL with a clear `file not found` message (existing behavior, now reachable).
- `should_skip_for_faas` was already a no-op (always `return 1`); removed.

## Tests

Shell syntax check (`bash -n`) on lib.sh, both `run.sh` files, and `start-backend.sh`: all pass.
