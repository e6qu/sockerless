# P86-005: Lambda live exec — agent-as-handler skeleton

**Status:** Done (skeleton; full AWS integration deferred)
**PR:** (pending — same branch)
**Branch:** `phase86-complete-runner-support`

## What

Laid the foundation for `docker exec` to work against Lambda-backed containers in live mode. The full end-to-end plumbing is blocked on access to a real `$AWS_LAMBDA_RUNTIME_API` endpoint, which is AWS-track work. This PR lands everything that can be built and unit-tested without live AWS:

- **Design doc** `docs/LAMBDA_EXEC_DESIGN.md` covering the overlay-image + bootstrap + reverse-agent architecture, including what still needs live verification.
- **`agent/cmd/sockerless-lambda-bootstrap`** — new binary, compile-clean, falls through to direct `syscall.Exec` of the user's entrypoint when `$AWS_LAMBDA_RUNTIME_API` is absent (local smoke tests). The Runtime-API loop is stubbed with a clear exit message and a TODO pointing at the AWS track.
- **`backends/lambda/image_inject.go`** — `RenderOverlayDockerfile(OverlayImageSpec)` produces the Dockerfile that layers the agent + bootstrap binaries onto the user's image and rewires `ENTRYPOINT`. Pure function, zero AWS dependency, unit-tested.
- **`backends/lambda/config.go`** — new `CallbackURL` field on `Config`, wired to `SOCKERLESS_CALLBACK_URL`.
- **`backends/lambda/integration_test.go`** — `TestMain` now runs unit tests when `SOCKERLESS_INTEGRATION != 1` (previously it just `os.Exit(0)`). All existing integration tests got a `skipIfNoIntegration(t)` guard so they skip cleanly. Same fix applied to `arithmetic_integration_test.go`. This was a latent blocker for any unit-test work in the lambda package.

## What stays blocked on AWS

- Bootstrap's Runtime-API loop (poll `/next`, co-process supervision, `/response` / `/error`).
- `image_inject.go` caller integration in `image_resolve.go` — needs a real `docker build` or registry-push path that the Lambda backend doesn't currently own (the existing ECR push fallback handles un-layered images).
- Reverse-agent registry on `BaseServer` (so `disconnectReverseAgent` from P86-004a can actually close a WebSocket).
- IAM policy guidance, VPC reachability, and security posture for the callback URL.
