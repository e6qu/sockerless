# P86-003: Services cross-container DNS via per-hostname Cloud Map services

**Status:** Done (AWS-track live verification deferred)
**PR:** (pending — same branch as P86-001/002)
**Branch:** `phase86-complete-runner-support`

## What

Refactored the ECS backend's Cloud Map integration so that cross-container DNS works on real Fargate. Previously all containers on a network were registered under a single Cloud Map service named `containers`, giving one DNS name per network. Now each container's hostname becomes its own Cloud Map service, and the ECS task definition sets `DnsSearchDomains` to the network's namespace so bare short-name lookups resolve.

## Files

| File | Change |

New `searchDomainsForContainer` helper for task-def DNS config. Old `findServiceInNamespace` removed |

## What this enables

CI runners that expose services via parallel `docker container create` on a shared network (GitLab docker-executor, GitHub Actions with `services:` / `container:`) can now reach each service by short name on real Fargate. Build container does `curl postgres:5432`; resolv.conf expands to `postgres.skls-<net>.local`; Cloud Map returns postgres's ENI IP.

## What still needs AWS

- Service-to-service security group ingress rules on the shared SG.
