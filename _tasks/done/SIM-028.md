# SIM-028: Update E2E variant routing + start-backend.sh for container backends

**Status:** DONE
**Effort:** S

## What Changed

Extended E2E infrastructure so container backends (ECS, CloudRun, ACA) get the same
treatment as FaaS backends in simulator mode:

- `lib.sh`: `uses_reverse_agent()` returns 0 for all 6 cloud backends. `get_test_variant()`
  uses this function (instead of `is_faas_backend`) for variant routing.
- `start-backend.sh`: Sets `SOCKERLESS_CALLBACK_URL` for all `uses_reverse_agent` backends
  in simulator mode (was previously FaaS-only via `is_faas_backend`).

## Files Modified

- `tests/e2e-live-tests/lib.sh` — Updated `get_test_variant()` to use `uses_reverse_agent`
- `tests/e2e-live-tests/start-backend.sh` — Changed callback URL condition to `uses_reverse_agent`
