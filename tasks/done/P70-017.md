# P70-017: Azure — Function App DefaultHostName Reachability

**Component:** SIM (simulators/azure)
**Phase:** 70 — Milestone 3 (Azure Fidelity)
**Depends on:** —
**Estimated effort:** S

---

## Description

Ensure `DefaultHostName` returned by Function App `BeginCreateOrUpdate` produces
a URL the backend can POST to for invocation. When running against the simulator,
this must resolve to `localhost:{port}`.

## Context

- Backend `backends/azure-functions/containers.go:210`: reads
  `result.Properties.DefaultHostName` for function URL
- Simulator `functions.go`: sets DefaultHostName from request Host header
- Invoke handler matches by Host header

## Acceptance Criteria

1. `BeginCreateOrUpdate` returns `DefaultHostName` that points to the simulator
2. Backend can construct `http://{DefaultHostName}/api/function` and reach
   the invoke handler
3. SDK test: create function app → extract DefaultHostName → POST to it →
   verify response
