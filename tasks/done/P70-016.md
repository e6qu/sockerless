## Description

Verify that ACA execution status returns the exact `JobExecutionRunningState`
enum strings the Azure SDK expects. The ACA backend checks for `Running`,
`Succeeded`, `Failed`, `Stopped`, `Degraded`, and `Processing`.

## Context

- Backend `backends/aca/containers.go:580,662,702`: checks execution status
  against SDK enum values
- SDK enum: `armappcontainers.JobExecutionRunningStateRunning`, etc.

## Acceptance Criteria

1. Execution status during active run is `"Running"` (exact string)
2. After successful completion: `"Succeeded"`
3. After stop: `"Stopped"`
4. SDK test creates execution → verify `Running` → stop → verify `Stopped`
5. SDK test covers `Succeeded` (auto-completion)
